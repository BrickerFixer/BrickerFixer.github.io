1. Основы языка Kotlin

1.1 Привет Мир!

Зачем сделали сам Kotlin

Kotlin — это современный статически типизированный язык программирования, который используется более чем 60% профессиональных Android-разработчиков. Kotlin помогает повысить производительность, удовлетворенность и безопасность кода. Kotlin был создан в 2011 году компанией JetBrains с целью сделать разработку на Java более выразительной, лаконичной и безопасной. Kotlin также обладает рядом преимуществ перед Java, таких как поддержка nullability в своей системе типов, совместимость с Java-кодом и библиотеками, поддержка функционального программирования и сопрограмм. В этом курсе вы научитесь разрабатывать Android-приложения на Kotlin, а также сравнивать его с Java и понимать его особенности и различия.

Kotlin был придуман, чтобы решить проблемы разработчиков JetBrains. Они нуждались в более простом инструменте, чем Java, для работы с их основным продуктом, называемым IntelliJ IDEA, полностью написанным на нем.

1. Ребята исследовали альтернативы, остались недовольны и решили придумать свой собственный язык. Kotlin был спроектирован как замена Java на операционной системе Android. Спустя восемь лет после его выпуска, в 2019 году, Google наконец согласился с Jemerov и большинством разработчиков Android и объявил, что Kotlin является предпочтительным языком для разработки приложений для Android2. Некоторые причины, по которым разработчики предпочитают Kotlin Java Kotlin более краток, экономя время, которое вы бы иначе потратили на написание шаблонного кода на Java. Вы можете преобразовать файл Java в файл Kotlin с помощью простого скрипта. Kotlin не имеет издержек во время выполнения. Иногда добавление функций к языку означает, что у него появляется больше издержек, которые снижают его производительность. Это не так с Kotlin. У Kotlin есть большое сообщество. Если вы застряли, вы можете легко найти других разработчиков, которые помогут вам на форумах и социальных сетях по кодированию. Kotlin упрощает асинхронное программирование. Совершение сетевых и баз данных вызовов асинхронно в Java неуклюже и болезненно. Kotlin имеет корутины, которые делают асинхронное программирование простым и эффективным. Kotlin обрабатывает nulls. Null в Java может вызвать сбой программы, если вы не подготовились к этому. В Kotlin вы можете добавить простой оператор к переменным, которые могут быть null, чтобы предотвратить эти сбои. Kotlin может работать на нескольких платформах. Kotlin может работать везде, где работает Java, поэтому вы можете использовать его для создания кросс-платформенных приложений. Перейти на Kotlin легко. Kotlin полностью совместим с Java, поэтому вам не нужно менять весь свой код сразу. Вы можете постепенно переводить приложение на использование Kotlin.
